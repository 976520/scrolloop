# 핵심 개념

scrolloop이 어떻게 수만 개의 데이터를 부드럽게 렌더링하는지 이해해 보세요.

## 윈도잉 (Windowing)

윈도잉이란 전체 리스트 중에서 **현재 사용자에게 보이는 부분(Window)**만 실제 DOM에 렌더링하는 기법입니다.

- **문제**: 10,000개의 아이템을 모두 렌더링하면 DOM 노드가 너무 많아져 브라우저가 느려집니다.
- **해결**: scrolloop은 스크롤 위치를 추적하여 현재 화면에 보이는 10~20개의 아이템만 렌더링합니다.

## 오버스캔 (Overscan)

스크롤을 아주 빠르게 할 때 흰 화면이 보이는 것을 방지하기 위해, 현재 보이는 영역 위아래로 몇 개의 아이템을 추가로 렌더링하는 기술입니다.

- scrolloop은 기본적으로 위아래 4개(`overscan={4}`)의 아이템을 더 그립니다.
- 리스트의 내용이 복잡하다면 이 값을 늘려 더 부드러운 스크롤 경험을 제공할 수 있습니다.

## 절대 좌표 배치 (Absolute Positioning)

scrolloop은 가상 리스트의 전체 높이를 유지하기 위해 내부 컨테이너를 실제 전체 데이터 높이만큼 키우고, 각 아이템을 `top` 좌표를 이용해 절대 좌표로 배치합니다.

이 방식은 각 아이템의 위치를 계산하기 매우 효율적이며, 브라우저의 레이아웃 재계산 비용을 최소화합니다.

## 성능 최적화 팁

1. **Memoization**: `renderItem` 함수에서 반환하는 컴포넌트를 `React.memo`로 감싸면 불필요한 리렌더링을 방지할 수 있습니다.
2. **Key 관리**: scrolloop 컴포넌트 내부에서 인덱스를 기반으로 기본 키를 관리하지만, `renderItem` 내부의 엘리먼트에도 적절한 키를 부여하는 것이 좋습니다.
